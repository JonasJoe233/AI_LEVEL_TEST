# AITI 设计决策记录

> 记录所有已拍板的设计决策及理由，新对话开始时优先读此文件。

## 产品定位

**AITI（AI Type Indicator）** —— 一个以 MBTI/SBTI 为参照物的病毒式 AI 水平测试工具。

- 目标受众：普通 AI 用户（非专业开发者），微信/小红书/朋友圈传播场景
- 核心价值：让人知道自己在哪里、笑一笑、顺手分享
- 原始灵感来源：卡兹克公众号文章《观察了三年，我把所有人用AI的水平分成了10个等级》
  → 完整原文已整理为 `AI使用水平10个等级.md`

## 四个核心维度

原文框架直接沿用，四维度不并行，各自独立进阶：

| 维度 | 英文代号 | 含义 | 测什么 |
|------|---------|------|--------|
| 可控性 | C | Controllability | 能不能驯服 AI，约束它精准产出 |
| 广度 | B | Breadth | 把 AI 用到多宽的领域 |
| 形态 | F | Form | 停留在 ChatBot 还是进阶到 Agent |
| 角色 | R | Role | 消费者还是创造者 |

## 题目设计

- **数量**：20道，每维度各5题
- **形式**：情景选择题（A/B/C 三选一）
- **排列**：四维度交叉排列（C,B,F,R,C,B,F,R...×5轮），避免连续同类题感
- **计分**：A=1分，B=2分，C=3分（暂无反向题，保持简单）
- **每维度总分**：5-15分 → 离散化为 L(≤8) / M(9-11) / H(≥12)

## 彩蛋设计原则

**不沿用 SBTI 的饮酒触发方式**，改为 **AI 异常检测风格**：

> 测试本身"表现得像 AI"——检测到异常答题模式，然后介入报告。

这一设计和"AI水平测试"的主题高度契合，也更有传播爆点。

## 技术方案

- **Vite 构建**：ES 模块，`npm run dev` 本地开发，`npm run build` 生成 `dist/`
- **模块结构**（数据与 UI 严格分离）：
  - `index.html`：纯 HTML 骨架，`<script type="module" src="/src/main.js">`
  - `src/main.js`：UI 逻辑（渲染、导航、分享卡），无硬编码数据
  - `src/engine.js`：计分纯函数（calcScores/getRating/getLevel/matchPersona 等），无 DOM
  - `src/data/questions.js`：20道题目 + 四维度名称/说明
  - `src/data/personas.js`：16个正常人设 + 3个彩蛋 + 稀有度数据
  - `src/styles.css`：全部样式
- **无后端**：零服务器，零数据库，答案不离开用户设备
- **分享卡**：html2canvas（npm 包，在 src/main.js 中 import）生成 PNG，直接下载
- **字体**：系统字体栈，无外部字体依赖
