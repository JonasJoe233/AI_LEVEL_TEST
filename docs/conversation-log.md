## 对话一（2026-05-11）

### 背景
用户（乔冠宇 / Jonas，百度文库/网盘 PM，负责 Oreate 和 Terabox 出海产品）希望基于卡兹克的《AI使用水平10个等级》文章，做一个类似 SBTI（MBTI 恶搞版）的 AI 水平测试工具。

### 逐步确认的决策

**Q1：结果页展示什么？**
→ 等级（Lv.0-10）+ 四维度得分 + 人设角色 + 稀有度

**Q2：人设怎么做？**
→ 基于互联网热门角色，有"稀有度"概念，博大家一笑，追求高流传度，无版权风险

**Q3：题目形式？**
→ 情景选择题（参考 SBTI 的场景化选项）

**Q4：SBTI 设计参考**
→ 用户提供了 SBTI 源码分析文章 + sbti-test-main 源码目录。
核心：纯前端规则匹配系统，人格描述静态写死，无AI调用。
详细研究见 `docs/sbti-research.md`

**Q5：测试叫什么名字？**
→ 用户否定了 AIBI，选定 **AITI（AI Type Indicator）**，理由：一眼看出在蹭 MBTI

**Q6：结果分享形式？**
→ 分享卡（Canvas 生成可下载图片）

**Q7：要不要彩蛋？**
→ 要，但触发机制必须结合 AI 测试背景，不能沿用 SBTI 的饮酒触发方式

**Q8（用户提案）：彩蛋机制方向**
→ 确认采用 **AI 异常检测风格**：测试"表现得像 AI"，检测到异常答题模式后介入
- GOD：全满分 → 逆向图灵测试
- CAVE：全极低 → 数字难民
- HYPE：角色高但可控性+形态低 → 嘴强王者

### 第一版开发完成
- 文件：`index.html`（单文件，纯前端）
- 20道题 + 16正常人设 + 3彩蛋
- 分享卡使用 html2canvas
