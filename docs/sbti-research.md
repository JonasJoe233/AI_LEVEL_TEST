# SBTI 参考研究笔记

> 来源：用户提供的源码分析文章 + 对 sbti-test-main/ 源码的直接阅读

## 核心机制（对本项目有参考价值）

### 整体架构
- **纯前端**，无后端服务器，无模型调用，答案停留在浏览器本地
- 使用 Vite 构建，但核心逻辑是静态规则匹配系统

### 两步出结果
1. **打分**：30道题 → 约15个维度 → 每维度累加 → 离散化为 H/M/L → 得到15位编码（如 `HML-LHM-MHL-HMM-LHL`）
2. **匹配**：25套预定义人格模板（同样是15位编码） → 逐一计算距离 → 选出最近的 → 输出对应人格

### 计分细节
- 大多数题：A=1分、B=2分、C=3分
- **有反向计分题**（A=3分、C=1分）——刻意打破直觉选择
- 每个维度通常由2道题组成

### 人格库
- **27套**人格（25常规 + HHHH兜底 + DRUNK隐藏）
- 每套：200-400字描述，**写死的静态文本**，不靠运行时生成
- 这是保证表达质量一致的关键设计决策

### 隐藏彩蛋
- **DRUNK 酒鬼**：题目中有饮酒习惯问题，选饮酒相关选项+极端回答触发，优先级高于常规匹配
- **HHHH 傻乐者**：所有25个模板匹配度都低（<60%）时触发，作为兜底

### 让它爆火的设计元素
1. **人格名字足够骚**：CTRL/ATM-er/Dior-s/SEXY/FUCK/DEAD……名字本身就是传播单元
2. **文案够冲够具体**：不说"您富有同理心"，说"你们家里有人叫你妈妈"
3. **天生适合截图**：结果页有专门设计的图卡，名字短，能秒懂
4. **结果有争议性**：测出 FUCK 草者/DEAD 死者 会让人想分享"我怎么测出这个"
5. **机制足够简单**：纯规则匹配，不依赖AI，传播时不需要解释技术

## 对 AITI 的启示

| SBTI 做法 | AITI 的对应选择 |
|-----------|----------------|
| 30题/15维 | 20题/4维（更聚焦，完成率更高） |
| 人格描述静态写死 | 同，所有描述预写好 |
| 隐藏彩蛋靠特定行为触发 | 改为 AI 异常检测风格（更切题） |
| 纯前端单文件可部署 | 同 |
| 人格名用英文缩写+中文绰号 | 同（CODE 工程师 / SOLO 一人军团）|
